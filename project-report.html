Executive Overview & Architecture

This section outlines the core vision of the project and the technological foundation chosen to achieve it. It provides context on what we are building and the tools powering the simulation environment.

🎯 Project Abstract

Designing and building a web-based network simulation tool inspired by Cisco Packet Tracer. The application features a GUI allowing users to create, manage, and visualize network devices (PCs, routers, switches) and their connections in a simulated environment. Key features include IP/MAC address assignment, subnet visualization, and monitoring device connectivity status.

🚀 System Goal

Deliver a responsive frontend with a navigation bar, interactive device sidebar, and a central visualization graph for nodes and edges. Users can interact dynamically (zoom, click-and-drag, manage ports). The backend logic securely processes data and topologies, providing reliable communication between the graphical interface and the database.

Technology Stack (v1.1.0)

💻

Frontend Architecture

Built with React (TypeScript / .tsx) for component management, styled efficiently using TailwindCSS v4. Graph visualizations are powered by Cytoscape.js. Network requests handled via Fetch API (JSON).

⚙️

Backend Server

Server logic, authentication, and API endpoints are driven by Rust (Cargo), replacing the previous Node.js setup to ensure high performance and strict memory safety during simulations.

🗄️

Database

User credentials, encrypted passwords, and serialized topology JSON structures are persistently stored in PostgreSQL 18.

Requirements Explorer

Interact with the tabs below to explore the detailed constraints and expected behaviors of the system. This modular view organizes the rules governing user interactions, core functionality, and system performance.

🛡️ Guest User

  • Presented with sign-up and sign-in pages upon loading.
  • Ability to create a new account and authenticate.
  • Restriction: Cannot interact with the simulation viewport until logged in.

🔑 Registered User

  • Full access to all webpage features upon login.
  • Can see and interact with the viewport topology.
  • Can create, add, name, and connect network device ports.
  • Can toggle device power (On/Off).
  • Can view and edit device details (IP, MAC, Subnet).
  • Ability to save the current network simulation state.
  • 1.
    Authentication: Registration and secure login system storing credentials.
  • 2.
    Device Management: Sidebar UI to add/delete devices. Auto-generation of MAC addresses, names, and IP subnet assignments.
  • 3.
    Topology Viewport: Graph node system reflecting live connections, updating dynamically on adds/removes.
  • 4.
    Connection Logic: Valid links: PC ↔ Switch, Router ↔ Switch, Router ↔ Router, Router ↔ PC. Invalid links: PC ↔ PC, Switch ↔ Switch.
  • 5.
    Port State: Ports handle single connections. UI updates to "CONNECTED". Offline devices temporarily remove visual connection edges.
  • 6.
    Simulation Engine: Animation of packet movement (small dot) along edges between established, online source/destination nodes.
  • 7.
    Persistence: Save and load functionality mapping topology to the authenticated user.

⚡ Performance

Graph updates must be instantaneous. Packet animations must be smooth. Backend operations optimized via native Rust execution.

🖱️ Usability

Modern, simple UI styled with TailwindCSS. Viewport supports zoom, pan, and click-and-drag interactions.

🛡️ Reliability

Strict enforcement of device connection logic to prevent duplicate or invalid port connections.

🌍 Compatibility & Env

Supports Chrome, Edge, Opera/GX, Firefox. Packaged for reliable, reproducible builds across developer environments.
